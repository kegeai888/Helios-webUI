# Helios 用户使用手册

> **项目地址**：https://github.com/kegeai888/Helios-webUI
> **原始项目**：https://github.com/PKU-YuanGroup/Helios

## 📖 项目简介

Helios 是一个实时长视频生成模型，支持三种生成模式：
- **T2V**（文本生成视频）：根据文字描述生成视频
- **I2V**（图片生成视频）：从静态图片生成动态视频
- **V2V**（视频生成视频）：对已有视频进行风格转换或内容变换

提供三个模型版本：
- **Helios-Base**：基础版本，质量最高
- **Helios-Mid**：中等版本，平衡质量与速度
- **Helios-Distilled**：蒸馏版本，速度最快（推荐日常使用）

## 🚀 快速开始

### 方式一：使用 Gradio Demo（最简单）

启动 Web 界面：
```bash
python app.py
```

打开浏览器访问显示的地址（通常是 `http://127.0.0.1:7860`），在界面中：
1. 选择生成模式（文本/图片/视频生成视频）
2. 输入提示词
3. 如果是 I2V 或 V2V，上传对应的图片或视频
4. 调整参数（分辨率、帧数、推理步数等）
5. 点击"生成"按钮

### 方式二：使用命令行脚本（推荐）

所有推理脚本都在 `scripts/inference/` 目录下，按模型和模式分类：

```bash
cd scripts/inference

# === Helios-Distilled（速度最快，推荐） ===
bash helios-distilled_t2v.sh    # 文本生成视频
bash helios-distilled_i2v.sh    # 图片生成视频
bash helios-distilled_v2v.sh    # 视频生成视频

# === Helios-Mid（平衡版本） ===
bash helios-mid_t2v.sh
bash helios-mid_i2v.sh
bash helios-mid_v2v.sh

# === Helios-Base（质量最高） ===
bash helios-base_t2v.sh
bash helios-base_i2v.sh
bash helios-base_v2v.sh
```

**提示**：脚本内已配置好默认参数，可直接运行。如需自定义，编辑对应脚本文件。

### 方式三：直接调用 Python 脚本

```bash
# 文本生成视频（T2V）
python infer_helios.py \
  --base_model_path ./models/Helios-Distilled \
  --sample_type t2v \
  --prompt "一只猫在草地上奔跑" \
  --height 384 \
  --width 640 \
  --num_frames 99 \
  --num_inference_steps 50 \
  --seed 42

# 图片生成视频（I2V）
python infer_helios.py \
  --base_model_path ./models/Helios-Distilled \
  --sample_type i2v \
  --prompt "让这张图片动起来" \
  --image_path ./example/image.jpg \
  --height 384 \
  --width 640 \
  --num_frames 99

# 视频生成视频（V2V）
python infer_helios.py \
  --base_model_path ./models/Helios-Distilled \
  --sample_type v2v \
  --prompt "将视频转换为水彩画风格" \
  --video_path ./example/video.mp4 \
  --height 384 \
  --width 640 \
  --num_frames 99
```

## ⚙️ 常用参数说明

### 基础参数
| 参数 | 说明 | 默认值 | 推荐范围 |
|------|------|--------|----------|
| `--base_model_path` | 模型路径 | `./models/Helios-Base` | - |
| `--sample_type` | 生成模式 | `t2v` | `t2v` / `i2v` / `v2v` |
| `--prompt` | 提示词 | - | 详细描述效果更好 |
| `--height` | 视频高度 | 384 | 256-768（需是 64 的倍数）|
| `--width` | 视频宽度 | 640 | 384-1280（需是 64 的倍数）|
| `--num_frames` | 生成帧数 | 99 | 33-297（越多越长）|
| `--fps` | 帧率 | 24 | 8-30 |
| `--num_inference_steps` | 推理步数 | 50 | 20-100（越多质量越好但越慢）|
| `--guidance_scale` | 引导强度 | 5.0 | 1.0-10.0（越高越贴合提示词）|
| `--seed` | 随机种子 | 42 | 任意整数（固定种子可复现结果）|

### I2V / V2V 专用参数
| 参数 | 说明 |
|------|------|
| `--image_path` | 输入图片路径（I2V 必需）|
| `--video_path` | 输入视频路径（V2V 必需）|

### 高级参数
| 参数 | 说明 | 适用场景 |
|------|------|----------|
| `--is_amplify_first_chunk` | 增强首帧质量 | 提升视频开头的清晰度 |
| `--is_enable_stage2` | 启用多阶段生成 | 生成超长视频时使用 |
| `--pyramid_num_inference_steps_list` | 多阶段推理步数 | 如 `20 20 20` |
| `--enable_compile` | 启用 torch.compile | 加速推理（首次运行会慢）|

## 💡 使用技巧

### 1. 提示词编写建议
- **详细描述**：包含主体、动作、环境、风格、光线等
- **好例子**：`一只橘色的猫在阳光明媚的草地上奔跑，背景是蓝天白云，电影级画质，4K高清`
- **差例子**：`猫跑`

### 2. 模型选择建议
- **日常使用**：Helios-Distilled（速度快，质量好）
- **追求质量**：Helios-Base（质量最高，但速度较慢）
- **平衡需求**：Helios-Mid（折中方案）

### 3. 分辨率与帧数建议
| 用途 | 分辨率 | 帧数 | 说明 |
|------|--------|------|------|
| 快速预览 | 384×640 | 33 | 1秒左右短视频 |
| 标准输出 | 384×640 | 99 | 4秒左右视频 |
| 高质量 | 512×896 | 99 | 更清晰但显存需求更高 |
| 超长视频 | 384×640 | 297 | 12秒左右，需启用 `--is_enable_stage2` |

### 4. 显存不足？
如果遇到 OOM（显存不足）错误，尝试：
```bash
# 方法一：使用 Group Offloading（降低显存至 ~6GB）
python infer_helios.py \
  --base_model_path ./models/Helios-Distilled \
  --sample_type t2v \
  --prompt "你的提示词" \
  --enable_group_offloading

# 方法二：降低分辨率或帧数
python infer_helios.py \
  --base_model_path ./models/Helios-Distilled \
  --sample_type t2v \
  --prompt "你的提示词" \
  --height 256 \
  --width 448 \
  --num_frames 33
```

### 5. 多卡并行推理
如果有多张 GPU，可以使用 Context Parallelism 加速：
```bash
# 使用 Ring Attention（2 卡）
torchrun --nproc_per_node=2 infer_helios.py \
  --base_model_path ./models/Helios-Distilled \
  --sample_type t2v \
  --prompt "你的提示词" \
  --enable_context_parallel \
  --context_parallel_type ring

# 支持的并行类型：ring / ulysses / unified / ulysses_anything
```

## 🎯 常见问题

### Q1: 生成的视频质量不理想？
- 增加 `--num_inference_steps`（如 50 → 80）
- 提高 `--guidance_scale`（如 5.0 → 7.0）
- 使用更详细的提示词
- 尝试更换模型版本（Distilled → Base）

### Q2: 生成速度太慢？
- 使用 Helios-Distilled 模型
- 降低 `--num_inference_steps`（如 50 → 30）
- 启用 `--enable_compile`（首次运行会编译，之后会快很多）
- 降低分辨率或帧数

### Q3: 视频出现闪烁或不连贯？
- 增加 `--num_inference_steps`
- 启用 `--is_amplify_first_chunk`
- 检查提示词是否包含冲突的描述

### Q4: 如何生成更长的视频？
- 增加 `--num_frames`（最大 297）
- 启用多阶段生成：`--is_enable_stage2 --pyramid_num_inference_steps_list 20 20 20`

### Q5: 如何复现之前的结果？
- 使用相同的 `--seed` 值
- 保持其他参数（分辨率、帧数、推理步数等）一致
