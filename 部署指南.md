# Helios 项目部署指南

## 快速开始

### 1. 环境准备

确保系统已安装 conda（anaconda 或 miniconda）。

### 2. 安装依赖

在 conda base 环境下安装项目依赖：

```bash
bash install.sh
```

该脚本会自动：
- 激活 conda base 环境
- 设置 HuggingFace 加速（http://hf.x-gpu.com）
- 安装所有依赖包
- 处理版本冲突

### 3. 下载模型

运行模型下载脚本：

```bash
bash download_models.sh
```

根据提示选择要下载的模型：
- **Helios-Base**: 基础模型
- **Helios-Mid**: 中等模型
- **Helios-Distilled**: 蒸馏模型（推荐用于 WebUI，速度快）

模型会下载到 `./models/` 目录。

### 4. 启动 WebUI

```bash
bash start_app.sh
```

该脚本会自动：
- 激活 conda base 环境
- 设置 HuggingFace 加速
- 检测并释放 7860 端口（如被占用）
- 启动 Gradio WebUI

访问地址：http://localhost:7860

## 手动下载模型

如果自动下载失败，可以手动下载：

```bash
# 安装 huggingface-cli
pip install "huggingface_hub[cli]"

# 设置加速
export HF_ENDPOINT=http://hf.x-gpu.com
export HF_HUB_DOWNLOAD_TIMEOUT=180

# 下载模型（以 Helios-Distilled 为例）
huggingface-cli download BestWishYsh/Helios-Distilled --local-dir ./models/Helios-Distilled
```

## 推理脚本使用

除了 WebUI，也可以使用命令行推理：

```bash
# 文本生成视频
bash scripts/inference/helios-distilled_t2v.sh

# 图片生成视频
bash scripts/inference/helios-distilled_i2v.sh

# 视频生成视频
bash scripts/inference/helios-distilled_v2v.sh
```


## 常见问题

### 1. 端口被占用

`start_app.sh` 会自动检测并释放 7860 端口，无需手动处理。

### 2. 模型下载慢

脚本已配置 HuggingFace 加速镜像（http://hf.x-gpu.com），如仍然很慢，可以尝试其他镜像：

```bash
export HF_ENDPOINT=https://hf-mirror.com
```

### 3. CUDA 内存不足

- 使用 Helios-Distilled 模型（显存占用更小）
- 减少生成帧数
- 降低分辨率
